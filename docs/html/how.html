A C++ library for computing the RO(G) homology of a point
Setting the Group Parameters

The parameters

  • For every group there are certain mandatory parameters that need to be set for the library to work. We will go over them in more detail below.
  • There are also some optional ones that allow you to identify and print the names of the computed Mackey functors. This functionality is disabled by default, but can easily be turned on by defining the macro MACKEY_NAMES and setting the optional parameters. The way this is done is explained in the file Optional_Implementation.h in the Demo folder here ... . The implementation there is for \(G=C_4\) and the Mackey functors in the \(RO(C_4)\) homology.

The mandatory parameters

We have included an example (for \(G=C_4\)) of how to set the group parameters in Implementation.h of the Demo folder (...). These parameters must be set before main starts. They all live in the GroupSpecific namespace.

Global variables

The global variables that need to be set are:

  • prime : the \(p\) in \(G=C_{p^n}\).
  • power: the \(n\) in \(G=C_{p^n}\).
  • reps : the number of nontrivial irreducible real representations of \(G\).
  • sphere_dimensions : the array consisting of the dimensions of those representations (so we must fix an order for them beforehand).

The standard differentials

  • There is one function that needs to be manually defined, and that's StandardDiff creating the differential for the standard chains. In practice what this means is setting the correct matrix given the input sphere. Apart from the case work that comes from math, I have made the construction of the differential as painless as possible.
  • This construction is done through the altmatrix function, that creates alternating matrices of the desired size and the desired "pattern". This pattern is repeated cyclically in the columns of the matrix.
  • An example: The matrix of size 4x4 with pattern \(a,b\) is
    \(\begin{matrix} a&b&a&b\\ b&a&b&a\\ a&b&a&b \\ b&a&b&a \end{matrix}\)
    If we use the pattern \(a,b,c,d\) instead we get
    \(\begin{matrix} a&b&c&d\\ b&c&d&a\\ c&d&a&b \\ d&a&b&c \end{matrix}\)
  • The matrices appearing in the Standard Chains all look like this due to equivariance.
